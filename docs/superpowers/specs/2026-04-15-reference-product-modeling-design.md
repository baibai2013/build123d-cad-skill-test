# build123d CAD 参考物建模工作流设计规范

**日期**：2026-04-15
**背景**：用户需要为已存在的产品（手机、芯片板、舵机等）建模配件（手机壳、安装座、散热壳等），需要先收集真实尺寸参数，再反推建模代码。
**目标**：在 build123d-cad skill 中新增「参考物建模」流程（Step R1~R5），做到：先告知搜索计划 → 用户确认 → 执行收集 → 汇总参数 → 进入标准建模。

---

## 触发条件

用户需求中包含**已存在的具体产品型号**，例如：
- 「帮我做红米 K80 的手机壳」
- 「给树莓派 4B 做一个散热壳」
- 「SG90 舵机安装座」
- 「HC-SR04 传感器支架」

---

## 完整流程（Step R1~R5）

```
Step R1: 识别产品 + 生成搜索计划（先告知，不执行）
    ↓ ✋ 用户确认 [ 开始搜集 ] 或 [ 我来提供资料 ]
Step R2: 执行资料收集（3来源 + 兜底）
    ↓ 写入 references/<product-slug>/
Step R3: 生成 params.md（参数汇总 + 置信度评级）
    ↓ ✋ 用户确认参数表
Step R4: 进入标准建模流程（单部件/多部件路由）
    ↓
Step R5: 收尾 — 提示用户是否删除 references/<product-slug>/
```

---

## Step R1：识别 + 搜索计划

AI 输出搜索计划，等用户确认（**先告知，不执行**）：

```
检测到目标产品：「<产品名>」
将按以下顺序收集参考资料：

📌 来源 1 — 官网规格页
  → <品牌>官网 <产品名> 产品页（尺寸 / 重量 / 摄像头 / 按键位置…）

📌 来源 2 — 购物平台
  → 京东「<产品名>」商品参数页
  → 淘宝「<产品名> 配件」同类产品标注尺寸

📌 来源 3 — 3D 模型库
  → GrabCAD 搜索「<产品名>」
  → Printables 搜索「<产品名> case」

收集完成后保存至：references/<product-slug>/
并生成 params.md 参数汇总表。

[ 确认开始搜集 ] [ 我来提供资料 ]
```

## Step R2：执行资料收集

### 文件夹结构

```
<用户工程目录>/
  references/
    <product-slug>/          ← 例如 redmi-k80、sg90-servo
      raw_specs.md           # 抓取的原始规格文本
      official_01.png        # 官方图片（正视/侧视/背视…）
      shopping_01.png        # 购物平台标注图
      model.step             # 如找到，下载的参考 3D 模型
      params.md              # ⭐ 汇总参数表（建模直接输入）
```

### 收集优先级

| 优先级 | 来源 | 置信度 |
|-------|------|--------|
| P1 | 官网规格页 | ★★★★★ |
| P2 | 京东/淘宝商品参数 | ★★★★ |
| P3 | GrabCAD/Printables STEP 模型 | ★★★（需验证） |
| P4 | 社区/论坛测量帖 | ★★★ |

### STEP 模型处理

找到 STEP 时，用 build123d 反推尺寸：

```python
from build123d import import_step
ref = import_step("references/<product-slug>/model.step")
bb = ref.bounding_box()
print(f"整体尺寸: {bb.size.X:.2f} x {bb.size.Y:.2f} x {bb.size.Z:.2f} mm")
# 进一步提取摄像头模组、按键等关键特征尺寸
```

### 兜底策略（找不到资料时）

```
未能找到「<产品名>」的完整尺寸资料。
请提供以下任意一种：
  [ ] 产品三视图（正/侧/背面照片）
  [ ] 官方规格截图
  [ ] 手动测量值（需要的话我提供测量位置清单）
```

## Step R3：生成 params.md

### 格式模板

```markdown
# 参数表：<产品名> 配件建模参考

## 数据来源
- 官网：<url>  ✅ 高置信度
- 购物平台：京东商品参数  ✅ 中置信度
- 3D模型：GrabCAD - <产品名>  ✅ 高置信度（已下载 model.step）

## 产品尺寸（建模直接使用）
| 参数         | 数值     | 来源       | 置信度 |
|-------------|---------|------------|--------|
| 机身长度     | ???mm   | 官网       | ★★★★★ |
| 机身宽度     | ???mm   | 官网       | ★★★★★ |
| 机身厚度     | ???mm   | 购物平台   | ★★★★  |
| 摄像头模组高度| ???mm  | 3D模型反推 | ★★★   |

## 配件建模参数建议
| 参数           | 推荐值  | 说明 |
|---------------|--------|------|
| 壁厚           | 1.2mm  | FDM 打印最小可靠壁厚 |
| 配合间隙（单侧）| +0.3mm | 确保装入顺畅 |
| 开孔余量（单侧）| +0.5mm | 防止遮挡关键接口 |
| 按键通孔高度   | +0.5mm | 高于机身按键面 |
```

**确认门 ✋** 用户确认参数表后才进入建模。

---

## Step R4：进入标准建模流程

以 `params.md` 为参数输入，按部件数量路由：
- **单个配件**（手机壳、保护套）→ 单部件 Step 1~4（含3变体OCP对比）
- **多件配件**（含支架、卡扣、盖板）→ 多部件 Phase 1~4

---

## Step R5：收尾提示

```
配件建模完成。
参考资料 references/<product-slug>/ 已完成使命，是否删除？
[ 删除 ] [ 保留作为设计依据 ]
```

---

## 与现有工作流的关系

本流程（Step R1~R5）是**建模前的数据准备阶段**，完成后无缝衔接现有工作流：

```
参考物建模流程 (R1~R5)
    ↓ 生成 params.md
单部件流程 (Step 1~4)  或  多部件流程 (Phase 1~4)
    ↓
STEP 导出 + OCP 预览 + 收尾
```

---

## 设计决策记录

| 决策 | 选项 | 理由 |
|------|------|------|
| 搜索执行时机 | 先告知计划，用户确认后执行 | 透明度优先，避免未经授权的网络请求 |
| 图片处理 | 下载到本地 | 链接可能失效，本地更稳定 |
| STEP 模型用途 | 提取参数反推代码 | 直接使用他人模型有版权风险，反推参数后自行建模更安全 |
| 资料清理 | 完成后提示用户选择保留/删除 | 用户决定，不自动删除防止误操作 |
| 找不到资料 | 提示用户提供三视图/实体图/参数 | 用户测量 > AI猜测，避免尺寸错误 |
